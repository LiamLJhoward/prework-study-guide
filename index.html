HTML

  • The head element contains information about the webpage
  • The body element represents the visible content shown to the user
  • Head and Meta elements are not shown on the website.
  • The Body section is shown.
  • < h1> is the main header, whilst < h2+> are the secondary headers.
  • element represents a paragraph or a block of text.
  • < ul>, < ol>, and < li> elements represent unordered lists, ordered list, and list items.
  • < img> elements contains information about images that are displayed. ‘Src’ contains the link to the image.
  • < br> element contains a line break

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • For when you start:
  • Git status = check the status of git
  • Git checkout main = Sends you to the main git
  • Git checkout -b feature/ = Sets up a new branch
  • You should see the following message in your terminal:
    • \
  • Switched to a new branch 'feature/add-html'
  • For when you’ve finished:
  • Git add -A
  • Git commit -m “type your message here”
  • Git pull origin main
  • Git push origin feature/

Javascript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
